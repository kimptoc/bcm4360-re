## Branch: BBPLL/HT Clock Not Coming Up

If BBPLL fails to enable HT clock after stage1:

**Root cause hypothesis:** `brcmf_chip_resetcore(d11)` leaves d11 in BCMA reset → d11 AXI fabric clock gated → firmware polls dead clock domain.

**Fix test (test.117)** — modified resetcore sequence:
1. `brcmf_chip_resetcore(d11, 0x000c, 0x0004, 0x0004)` (current: FGC|CLK → clear RESET)
2. Verify via wrapper: RESET_CTL=0, IOCTL has CLK=YES
3. Read d11.clk_ctl_st — should show HAVEHT=YES, BP_ON_HT=YES if BBPLL is up
4. THEN ARM release

If test.117 succeeds (firmware passes fn 0x1415c), keep this sequence.
