  CRITICAL FINDING: 0x43b1 may NOT be a counter
======================================================================

Function at 0x673cc:
  MOVW R0, #0x43b1
  BX   LR

This function returns the CONSTANT 0x43b1 — the SAME value that TCM[0x9d000]
"freezes" at. This raises the possibility that:

1. TCM[0x9d000] is set ONCE during init to 0x43b1 (a static value, not a counter)
2. If true, the WFI-disproof is INVALID (was based on "frozen counter = CPU halted")
3. Firmware may have completed init normally and be sitting in WFI idle
4. The "hang" may actually be: firmware waiting for PCI-CDC commands the host never sends

To resolve: test.89 must sample TCM[0x9d000] at high frequency (~1ms intervals)
during the first 100-200ms after ARM release to see if intermediate values exist
(0, 1, 2, ..., 0x43b1) or if it jumps straight from 0 to 0x43b1.
