### 0x644DC: blx r3 (Call 1 via vtable[+4])
r3 = VT[0x58cf4] = 0x1FC3 → calls 0x1FC2 (Thumb)

### 0x1FC2 (VT function = TRAMPOLINE, 6 instructions):
```
mov r2, r1              ; r2 = arg1 (originally r1)
```

→ 0x1FC2 rearranges args and calls 0x2208 (no stack frame, pure tail call)

### 0x1FB4 (vtable[+8] = similar trampoline):

### 0x1F78 (vtable[+12] = actual function):

### 0x2208 (THE REAL INIT FUNCTION):
```
push.w {r0,r1,r4,r5,r6,r7,r8,lr}  ; 8 regs (32 bytes)
r7 = *0x232C = 0x00062A14          ; global state ptr
r4 = arg0 (obj)
r6 = arg1
r5 = arg2
r3 = *0x62A14 = 0x58CF0            ; vtable ptr (global state)
if (r3 & 2): debug print
if (r6==0 || r5==0): error path
...
r8 = 0
r0 = obj->field4
r1 = obj
r2 = obj->field0
r3 = r6
bl 0x1FD0               ; allocate 76-byte struct
obj->field12 = r0
if (r0 == 0): error path
r0 = *0x62A14 & 0x10   ; test bit 4
if (bit4 == 0): goto return (0x231C)

r0 = 0

    → calls 0xa4c, then branches to 0x848
```

### Key: bit4 of *0x62A14 IS set:
*0x62A14 = 0x58CF0 (from test.93)
0x58CF0 & 0x10 = 0x10 (non-zero) → bit4 IS set!
→ The code ALWAYS calls 0x5250 for this chip
